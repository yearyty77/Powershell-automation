### Description ###
The application retrieves approved access request tickets from ServiceNow and uses the request details to create and configure user accounts in Centric PLM through its API. After successful user creation, the originating ServiceNow tickets are updated and closed automatically. This approach helps standardize the provisioning process, reduce errors, and shorten the time required to onboard new users.
Key Capabilities

### Key Capabilities ###
- Retrieves and processes approved access request tickets from ServiceNow
- Creates and configures new users in Centric PLM using API calls
- Updates ticket status and closes requests in ServiceNow upon completion

### Purpose ###
The primary goal of this project is to streamline access management workflows, reduce repetitive manual tasks for administrators, and ensure consistent and auditable user provisioning across systems.
